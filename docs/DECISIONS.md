# Architecture Decision Records (ADR)

This document records architectural decisions made in this project.

## Format

Each ADR follows this format:

- **Status**: [Proposed | Accepted | Deprecated | Superseded]
- **Context**: What is the issue we're addressing?
- **Decision**: What is the change we're proposing or have agreed to?
- **Consequences**: What becomes easier or more difficult?

## ADR-001: Project Structure

**Status**: Accepted

**Context**: Need a standardized project structure for team collaboration.

**Decision**: Use a modular structure with separate directories for source, tests, docs, and scripts.

**Consequences**:
- Easier navigation and onboarding
- Clear separation of concerns
- Consistent across projects
